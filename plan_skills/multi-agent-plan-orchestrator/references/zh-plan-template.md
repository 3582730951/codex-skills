# 中文计划模板

在 Codex 写计划、进入 `/plan`、或重新规划时，优先按这个模板输出。

```text
执行模式:
- real-multi-agent | constrained-single-agent

保障级别:
- tiny | standard | heavy

任务章程:
- 目标:
- 范围:
- 非目标:
- 验收标准:
- 约束:
- 禁止走捷径项:
- 事实来源:

主 PM:
- main_agent

Agent 编组:
- PM:
- 架构师:
- 工程师:
- 测试:
- 架构审查:
- 安全审查:
- 完整性审计:

委派映射:
- 哪些是实际分配出去的 agent
- 哪些只是受限回退检查点
- 如果不能真实分配，为什么不能

架构合同摘要:
- 边界:
- 接口/数据契约:
- 不变量:
- 任务专属编码规范:
- 风险点:

并行轨道:
- 轨道名 / owner / 依赖 / 产物 / 禁止直接改动的文件

文件所有权:
- 文件或模块 -> owner

模型路由:
- 角色 -> tier -> 具体模型或别名 -> 原因

上下文包:
- 每个角色允许看到什么
- 每个角色不能依赖什么

质量门禁:
- 测试门禁:
- 架构门禁:
- 安全门禁:
- 独立审核门禁:

保障评分卡:
- 架构完整度:
- 证据覆盖度:
- 审核独立性:
- 上下文完整性:
- 任务连续性:
- 安全覆盖度:
- 委派完整性:
- Gate Result:

合并顺序:
- 第一步:
- 第二步:

开放风险:
- 风险 / owner / 缓解措施

状态快照:
- 当前事实:
- 已接受决策:
- 被否决路径:
- 阻塞项:
- 下一步:
```

如果运行时支持真实子 agent，`standard` 和 `heavy` 计划必须把关键角色真实分配出去，不能只由 `main_agent` 单线程扮演所有人。
